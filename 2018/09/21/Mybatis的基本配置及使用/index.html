Mybatis的基本配置及使用

Created2018-09-21|Updated2019-10-20
|Post View:

导入Maven依赖

配置XML文件

配置Mybatis我们需要配置2个XML文件,一个是全局配置文件,这个配置文件主要配置的是mybatis的运行必要的配置，包括数据源(dataSource),加载三方插件(plugins),设置别名,设置映射文件的路径等等。另外一个就是局部映射文件，这个配置文件主要是用来书写每个POJO的(Plain Ordinary Java Object )SQL语句。

全局配置文件的配置

1. 引入约束:
2. 配置相关的node节点
  • properties
    在这里面可以配置一些mybatis需要配置的配置项数据
    eg: 在下面的其他节点中，你可以用 **${}*来引用其他的节点
    eg: 当项目中多个地方存在对同一属性配置的时候，Mybatis的加载顺序是：
  1. properties主体中配置的property
  2. classpath下的properties下的文件
  3. sqlSessionFactory.build() 方法中配置的property

    tips: 后加载的配置会覆盖先前加载的配置

  • setting
    这个节点主要约束了mybatis在运行时的行为，具体的配置请参阅
    http://pye1st3o7.bkt.clouddn.com/mybatis-settings.xlsx

  • typeAliases
    可以为Java类型起别名，而不必使用权限类名 例如在你要使用Integer类型的时候，比就不必在写java.lang.Integer,而直接就可以用你声明的别名来代替。当然在Mybatis中8大基本类型都有默认别名，不必在进行设置,这里只是那他来举个例子.
    你也可以使用package来指定要起别名的包

    我们也可以使用 @Alias 注解的方式来给Java对象起别名

  • objectFactory
    这个节点用来生成mybatis返回结果集的，它有两个方法一个用来处理无参构造器，一个用来处理有参有参构造器。

  • plugins
    这个节点主要是用来配置mybatis的插件，例如分页插件pageHandler,Executor,ParameterHandler,ResultSetHandler,StatementHandler等等。\

Note: plugins中的方法可能会修改Mybatis本身的行为 所以使用时要小心。

  • envenoments
    eg:

这个标签主要配置mybatis运行时所需的环境,这个环境可以配置多个，但每次只有一个环境生效，你可以在envenoment里指定id属性来表示不同的环境,也可以在envenoments中设置default属性设置默认的运行环境。同时你可以在SqlSessionFactory.build(render,envenoment) 方法中设置生效的环境,若没设置则会使用default指定的环境。
transactionManager指定事务的处理方式,有两种方式可供选择

类型 JDBC MANAGED
处理事务的方式 使用JDBC的commit和rollback机制，依赖数据库连接来管理事务 不会做任何事，不会commit也不会rollback，仅仅只是让容器本身来管理事务(eg:JEE Application Server Context),默认的行为是关闭此次连接。

Note:你可以通过使用如下配置来关闭默认行为

datasource是用来设置mybatis所使用的数据源，type属性用来是指定数据源的种类

UNPOOLED POOLED JNDI
不使用连接池 适用于小型应用 使用连接池,最常见的配置 仅仅适用于EBJ和Application Server场合
  • dataScourceProvider
    指定数据库方言,这也是Mybatis能执行多种数据库的SQL语句的原因。
  • mappers
    指定mapper映射文件的路径,有三种配置方式

1.url
<mapper url="file:///var/mappers/AuthorMapper.xml"/> 不常用

2.class
<mapper class="org.mybatis.builder.AuthorMapper"/>
3.resource
<mapper resource="org/mybatis/builder/AuthorMapper.xml"/>

我们同样可以来指定要扫描的包名
<package name="org.mybatis.builder"/>

Mapper映射文件的配置

1.导入约束
2.常用节点
  • select
    主要书写查询类的SQL语句

此处的 id 对应映射接口中的方法名, parameterType 为入参的类型，resultType 为返回结果的类型。
我们可以使用**#{}**来使statement转化为preparedStatement,#{id}在运行时将会被解析成?。

  • update
    主要书写更新操作用的SQL语句

  • delete
    主要书写删除用的SQL语句

  • insert
    用于插入操作的SQL语句

NOTE: 如果在数据库表中有自动生成的字段时，我们可以使用useGenerateedKey=true配置

在日常开发中,我们可能有的时候仅仅只需要插入一条数据,但有时我们往往会要求返回插入数据的Id，此时在进行查询操作就不合适了,效率也会变低,这是我们可以使用Mybatis提供的selectKey，他可以返回我们的自动生成的id

当然 <insert id="insertStudent" parameterType="Student" useGenerateedKey=true keyProperty="id">也可以实现同样的效果
获取的id用student.getId() 接收就行了。

  • sql 片段
    可以用其来来替换掉里面的sql语句,实现了sql的复用

    使用<include refid="xx"></include>来引用对应的代码块

  • ${}的使用

**${}**是一种字符串的匹配,他会把解析的数据直接渲染成字符串，
而不是像 #{} 一样用?占位,这在某些场合，例如模糊查询时很有用,
但是它是不安全的，由于它会直接渲染数据 所以就可能导致 sql注入 ,使用时一定要小心。

mybatis的配置

Mybatis其实核心就三大块，有两个我们已经讨论过了，就剩下最后一个组件SqlSessionFactory。SqlSessionFactory加载全局配置文件并创建SqlSession 会话对数据库进行CRUD操作。

Mybatis官方提供两种方式来创建这个SqlSessionFactory对象

  • 一种是通过InputStream流的方式来加载
  • 还有一种是通过Configration对象来创建 至此,Mybatis的基本使用及配置就介绍完了,感谢各位的捧场。
