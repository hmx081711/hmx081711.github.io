使用springMVC开发Restful API

Created2019-09-30|Updated2019-10-20
|Post View:

第一印象

操作 传统web应用 Rest风格web应用
查询 localhost:8080/query?username=”小明” localhost:8080/query/1
创建 localhost:8080/create?username=”小明”&age=16… localhost:8080/create/user
修改 localhost:8080/update?id=1 localhost:8080/update/1
删除 localhost:8080/delete?id=1 lcoalhost:8080/delete/1

Restful风格Web应用的特点

1.使用url来描述资源
2.使用http方法来描述行为
3.使用http状态码来描述结果
4.使用json来交互数据
Note: Restful只是一种风格，并不强制要求实现

编写第一个Restful API

  • 搭建Web环境，这里我们使用MockMvc来模拟创建web环境 这件就搭建好了模拟好的Web环境
  • 编写测试用的Controller

GetMapping注解是RestApi中的一个重要的注解，它相当于 @RequestMapping(value="targetUrl",method="RequestMethod.GET"),与其相对应的注解还有PostMapping,DelateMapping,PutMapping,其原理都和GetMapping一样。

  • 编写针对Restful的测试用例

使用MockRequestBuilders来发起http请求,并设置请求的contentType,这里选择的是Json格式返回。andExcept()方法是设置期待的结果。andReturn()方法是用来获取返回的结果,并按照指定格式来处理返回结果。

使用注解申明Restful API

常用注解

  • @RestController
    标识该应用是Rest风格的，标识为此注解的类中的所有Http方法若要返回Json数据时,不必再为它标注@ReponseBody注解
  • @JsonView
    这个注解是用来在不同的请求中显示指定的字段信息，因为在某些情况下我们不需要返回对象的所有信息，或者不应该返回对象的所有信息。
    下面是使用JsonView注解的具体操作:

  1. 定义视图接口，这个接口表明了该对象返回的不同状态

    这里就定义了两个返回视图,返回的时候User对象的状态就有两种显示方式

  2. 在需要特定处理的字段或方法上标注@JsonView，并指明value属性

    我们可以看到User对象中的password的字段被标记成了 @JsonView(UserDetailView.class)，这表示在Controller中声明为@JsonView(UserDetailView.class)的http方法中才包含password的信息，由于
    UserDetailView继承了SimpleUserView，所以在UserDetailView中也会显示username字段。

  3. 在Controller中的http方法上标识要显示的视图

    这样就可以按要求返回相应的视图了。

  • Valid
    此注解是用来进行表单验证的，标注为@Valid的对象,会在请求发起之后，根据在对象字段上定义的一些验证注解进行表单校验。校验的处理结果被封装到了BindingResult对象中,该对象允许程序在校验失败后不抛出异常，任然进入http方法中运行。 从bindRequest对象中可以去到每个字段的错误信息,并进行处理。 这些注解都是验证用的注解，关于这些注解还有很多，具体可以查阅org.hibernate.validator.constraints,javax.validation.constraints等包。
