什么是Http

Http全称超文本传输协议，是应用层的协议,是一种基于请求和响应,无状态的协议。下面就从请求头和响应头来介绍HTTP。

  1. RequestHander
请求头字段 作用
Accept 用于告诉服务器，客户机支持的数据类型
Accept-Charset 用于告诉服务器，客户机采用的编码
Accept-Encoding 用于告诉服务器，客户机支持数据压缩格式
Accept-Language 客户机的语言环境
Host 客户机通过这个头告诉服务器，想访问的主机名
If-Modified-Since 客户机通过这个头告诉服务器，资源的缓存时间
Refer 客户机通过这个头告诉服务器，它是从哪个资源访问服务器的(防盗链)
User-Agent 客户机通过这个头告诉服务器，客户机的软件环境
Cookie 客户机通过这个头向服务器带数据
Connection 这个请求完了，是保持连接还是关闭
Range 断点下载 
    1. bytes=n1-n2,传输范围n1到n2字节
    2. bytes=n-，传输web资源中第n个字节以后的所有内容
    3. bytes=n,传输最后n个字节
  1. ResponseHandler
响应头字段 作用
Server 服务器通过这个头，告诉浏览器服务器的类型
Content-Encoding 服务器通过这个头，数据的压缩格式 (相关java知识:GZIPOutputStream,包装流/底层流)
Content-Length 服务器通过这个头，告诉浏览器回送数据的长度
Content-Type 服务器通过这个头，告诉浏览器回送数据的类型
Last-Modified 服务器通过这个头，告诉浏览器当前资源的缓存时间
Refresh 服务器通过这个头，告诉浏览器隔多长时间刷新一次
Content-Disposition 服务器通过这个头，告诉浏览器以下载方式打开
Transfer-Encoding 服务器通过这个头，告诉浏览器数据的传送格式
Etag 缓存相关的头部，用于实时性要求高的系统
Expires 服务器通过这个头，告诉浏览器把回送的资源缓存多长时间，-1或0则不缓存
Cache-Control和Pragma:no-cache 服务器通过这两个头，也是控制浏览器不要缓存数据
Connection 断开连接/保持连接
Date 当前时间
Accept-Ranges 用来说明web服务器是否支持range。支持返回bytes;不支持返回none
Content-Range 制定了返回web资源的字节范围，格式：n1-n2/n_total

tomcat简介

Tomcat是Apache 软件基金会（Apache Software Foundation）的Jakarta 项目中的一个核心项目，由Apache、Sun 和其他一些公司及个人共同开发而成。Tomcat服务器是一个免费的开放源代码的Web 应用服务器，属于轻量级应用服务器，在中小型系统和并发访问用户不是很多的场合下被普遍使用，是开发和调试JSP 程序的首选。

tomcat的目录

  • bin : 包含了tomcat在各大平台上的运行启动/关闭文件
  • conf: 包含了tomcat运行时的必要的一些配置文件,如server.xml
  • work: 存放Jsp编译后产生的class文件
  • webapps: web应用部署的文件夹
  • lib: tomcat运行时必须的一些jar包
  • logs: tomcat运行时的日志文件
  • temp: tomcat在运行时的临时文件

    tomcat的配置

    从apache-tomcat官网直接进行下载,解压后不需要进行任何配置,进入bin,根据不同环境选择不同的文件，即可直接运行,默认的运行端口号为8080,如若要修改运行的端口号只要修改conf/server.xml即可实现

什么是JavaWeb

Java Web，是用Java技术来解决相关web互联网领域的技术总和。web包括：web服务器和web客户端两部分。Java在客户端的应用有java applet，不过使用得很少，Java在服务器端的应用非常的丰富，比如Servlet，JSP和第三方框架等等。Java技术对Web领域的发展注入了强大的动力。

创建JavaWeb项目

  1. 打开IDEA,依次点击File/Project
  2. 选择maven项目
  3. 设置maven项目工程的坐标
  4. 设置项目名,生成maven项目
  5. 点击File/ProjectStruct
  6. 选择Faces，Web 在弹出的选项框中选中当前项目
  7. 设置web.xml的生成路径
  8. 设置web项目的路径
  9. 完成后发现项目中生成了webapp的文件夹,至此web工程就搭建完了

    简单来说Servlet就是运行在应用服务器上的程序,是处理客户端发来的Http请求的工具。是连接浏览器和服务器的桥梁。

Servlet中常见的对象

  • ServletContext
    Servlet的上下文对象,一个tomcat内的所有Servlet共享一个ServletContext对象,可以实现数据共享，应此ServletContext对象也被称为Context域对象,作用范围为整个应用。
    下面是ServletContext的常见用法:

  1. ServletContext也可以通过 getResourceAsStream() 方法，来加载配置文件

  2. 可以通过ServletContext实现转发
    this.getServletContext().getRequestDispatcher("/the url").forword(req,res);

  3. 可以为web应用添加Filter，Listener

  • ServletConfig
    Servlet的配置对象,可以通过web.xml中的<servlet>标签中
    <init-param>标签来配置Servlet,可以设置输出的字符集,和指定加载的配置文件

  • HttpServletRequest
    HttpServletRequest是浏览器发过来的请求对象,它的主要作用是申请访问互联网上的某些资源或服务。我们可以通过Request对象获取请求的参数，方法，请求的URI等信息，还可以获取请求客户机的IP地址等信息，下面是具体的获取方法。

方法名 作用
getParameter(String s) 获取请求的参数
getRemoteAddr() 获取请求的客户机的IP地址
getRequestURI() 获取请求的URI
getMethod() 获取请求的方法
  • HttpServletResponse
    HttpServletResponse是服务器包装的要返回给客户端的响应对象，我们可以通过Response对象向页面上输出一些信息，还可以使用Request对象对页面实现重定向,
    还可以设置响应的ContentType,字符编码等。下面是具体的获取方法。
方法名 作用
getWriter()/getOutputStream() 获取页面输出的流对象
sendRedirect(String url) 重定向到指定页面
setContentType() 设置响应的内容类型
setCharacterEncoding() 设置响应的字符编码
setHeader(String s1,String s2) 设置响应头

Servlet的生命周期

从servlet的方法init(),service(),destroy()来看servlet的生命周期大致分为
创建,初始化,运行,销毁四个部分，这四个中的三个状态分别对应着上述的三个方法。
NOTE:destroy()方法只会在移除Servlet容器的时候才会调用，并且只会调用一次。

Jsp

什么是Jsp

Jsp是一种动态网页技术，它和Servlet一样可以根据请求响应相关的HTML，XML页面给客户端，它支持在HTML页面中写Java代码,极大地拓展了Servlet。JSP首次编译时会生成Servlet的.java,.class文件存放在tomcat的work目录下,可以说JSP是一种特殊的Servlet。

Jsp的渲染过程

这个问题我们只要进入tomcat的work目录下一看究竟就知道怎么回事了

这是我事先准备编译好的的JSP文件,我们可以发现HTML语句被out.write()语句渲染到了页面上的,这个out是什么呢?，其实它是Jsp的9大内置对象之一，接下来会详细介绍。

JSP指令

Jsp的指令主要有三个

  • page指令
    <%@ page %> 这个指令主要是用来设置jsp页面的信息，例如字符编码，ContentType
  • include指令
    <%@ include %>
    这个指令主要是用来引用其他的jsp页面的,一般来说是页头，页脚。
  • tablib指令
    这个指令主要是用来引用一些三方的标签库。

JSP的动作标签

  • <% %>
    可以在里面写Java代码，可以嵌套使用
  • <%= %>
    可以计算一些表达式的值,不需要加;号
  • <%@ %>
    引用JSP指令

JSP的9大内置对象

名称 用途 Servlet中的对应
out 向页面输出一些信息 response.getWrite()
request 用户的请求 HttpServletRequest对象
response 服务器的响应 HttpServletResponse对象
session 获取session域 session
pageContext 获取其他8大内置对象
page 整个Jsp页面 this
exception jsp页面异常,只有该页面被申明为错误页面时才会显示 Throwable
config jsp的配置信息 contextConfig
application 整个应用 ServletContext

Filter

什么是Filter

Filter又称过滤器,是JavaWeb一种很重要的技术,开发者可以通过Filter对Web服务器上的资源进行管理，权限控制等操作。

Filter的工作流程


我们可以看到web浏览器向Web服务器发送请求,请求会通过filter拦截匹配的url进行验证,验证通过filter会调用 filterChain.dofilter() 进行放行,若该web容器内没有其他匹配的filter的话,就会访问web资源,并包装response对象返回，否则将去往下一个filter继续进行验证,当所有的filter都通过时,才能访问资源。

第一个Filter的程序

要将一个类声明为filter很简单,只需要实现HttpFilter接或继承GenericFilter就行了

和Servlet类似,Filter也需要注册才能生效

当然也能使用注解进行注册

Filter的生命周期

Filter的生命周期和Servlet相似,分为四部分创建,初始化,拦截,销毁,由Web容器管理,当应用启动时创建Filter对象,并调用init()方法进行初始化,初始化操作仅仅进行一次，当有请求来时进行拦截,当web容器调用destory()
方法时进行销毁,”销毁操作仅进行一次”。

Listener

什么是监听器

Listener又名监听器,是JavaWeb技术重要的一环,主要用来监听Web应用的事件,行为。

监听器的分类

  • 按域对象分
名称 作用
ServletContextListener 监听Context对象的创建和销毁
HttpSessionListener 监听Session对象的创建和销毁
ServletRequestListener 监听Request对象的创建和销毁
  • 属性监听器
名称 作用
ServletContextAttributeListener 监听Context域属性的创建,销毁，和更新
HttpSessionAttributeListener 监听Session对象的属性的创建,销毁，和更新
ServletRequestAttributeListener 监听Request对象的属性的创建,销毁，和更新
  • HttpSessionBindingListener
    这个监听器和其他的监听器不同,它是声明在实体类上的，当Session域绑定或解绑该对象的时候，该监听器触发。
